
> Тут описаны нефункциональные требования. Ожидаемая производительность, надежность и другая фабула.

| **Критерий**                          | **Расшифровка**                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Производительность и отзывчивость** | <ul><br><li> **Отклик API:** Время ответа сервера на запросы чтения (GET `/api/tasks`) не должно превышать **500 мс** при нормальной нагрузке. </li><br><li> **Отклик UI:** Рендеринг списка задач (до 100 элементов) на клиенте должен занимать не более **200 мс**. </li><br><li> **SPA Navigation:** Переход между разделами (Dashboard -> Kanban) должен происходить мгновенно (< 100 мс) без перезагрузки страницы. </li><br></ul> |
| **Надёжность**                        | <ul><br><li> **Обработка ошибок:** При падении API или отсутствии сети клиентское приложение не должно "падать" (белый экран). Должно отображаться уведомление (Toast) или заглушка ошибки. </li><br><li> **Валидация данных:** Все формы (Login, Create Task) должны иметь валидацию на клиенте и на сервере. </li><br></ul>                                                                                                           |
| **Хранение данных**                   | <ul><br><li> **СУБД:** PostgreSQL 16+. </li><br><li> **ORM:** Entity Framework Core (Code First подход). </li><br><li> **State:** Состояние интерфейса (выбранный проект, открытая вкладка) не сохраняется при закрытии вкладки браузера, но сохраняется токен авторизации (JWT). </li><br></ul>                                                                                                                                        |
| **Ограничения платформы**             | <ul><br><li> **Бэкенд:** ASP.NET Core 10.0 (C#). </li><br><li> **Фронтенд:** React 18 + Vite. </li><br><li> **Браузеры:** Поддержка последних версий Chrome, Edge, Firefox, Safari. </li><br><li> **Экран:** Оптимизация под Desktop. </li><br></ul>                                                                                                                                                                                    |
| **Безопасность**                      | <ul><br><li> **Аутентификация:** Использование JWT (Access Token). Токен передается в заголовке `Authorization: Bearer ...`. </li><br><li> **Пароли:** Хранение паролей в БД только в хэшированном виде (используя Identity или BCrypt). </li><br><li> **CORS:** API должно принимать запросы только с разрешенных доменов клиента (localhost в разработке). </li><br></ul>                                                             |
